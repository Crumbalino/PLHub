Core Principles
No gambling content — ever. Hardcoded rule. No gambling ads, sponsorship, or branded content. This is non-negotiable.
No clickbait. Headlines are clear and informative. AI summaries give you the story in seconds.
ADHD-friendly UX. Short-form, scannable, rewarding. High contrast, large fonts, bite-sized AI summaries. Get what you need and move on.
Community-ranked. Pulse scores surface what fans care about, not what editors push.

Global Transition Rules
Duration: 200ms for interactive elements, 300ms for page-level colour changes (dark/light toggle).
Easing: ease-out for enters, ease-in for exits. Never linear — it feels mechanical.
Transform origin: Centre for scale, top-left for reveals.
Principle: Every interaction should feel premium and rewarding. Smooth transitions, gold accent for feedback, no jarring snaps. The site should feel like a nice place to be.

ApproachCSS variables on :root
Class.dark (default) / .light
No flashScript in <head> reads pref
Implementation: Use CSS custom properties toggled by a class on <html>. A blocking <script> in <head> reads localStorage before paint to prevent flash-of-wrong-theme (FOWT). The toggle updates the class and persists the preference.

Future Features
Scoped but not yet built. Design decisions captured here so they're consistent when implemented.
Scoped
The Snapshot Ticker
Horizontal ticker bar below the nav. Shows live scores, p.100 page number, "Today: X matches". Monospace scores. Teal accents on page numbers and match count. Pink for LIVE indicator with pulsing dot. Background: slightly darker than page (rgba black overlay). Auto-scrolls on mobile, full-width on desktop.
Scoped
AI Pundit Characters
Multiple named characters with distinct voices. The Secret Pundit (sharp, opinionated) already exists as "SP" badge. Future: tactical analyst, stats nerd, hot-take merchant, fan voice. Each gets a unique 2-letter badge and distinct colour from the palette. Characters defined in tone-of-voice work, visual treatment defined here.
Scoped
User Profiles / Auth
Sign in via the nav bar (currently placeholder). Profile page will follow card design language — navy card background, teal structural elements. Avatar placeholder: diagonal frame icon. User preferences: favourite club, dark/light default, notification settings. Authentication via Supabase Auth.
Scoped
Notification Badges
Small pink dot (8px) for unread counts on nav items. Larger pink pill with white number for counts > 9. Position: top-right of parent element, offset by -4px. Pulse animation on new notifications (same as LIVE dot). Use sparingly — this is not a social media app.

The Teletext Thread
Not a theme. Not cosplay. A shared cultural memory baked subtly into the design language.
The teal-on-navy palette already echoes Ceefax's cyan-on-dark aesthetic — this happened organically. The following elements make it intentional without making it a pastiche.
p.100 Page Numbers
The Snapshot bar uses "p.100" — a direct Ceefax reference. Subtle enough that under-25s think it's a design choice. Over-25s feel it instantly.
Block Reveal
StoryCards load with a quick blocky reveal — content appears in horizontal strips, like a page rendering. CSS only, no performance cost.
Monospace Scores
Live scores in the ticker use monospace type. The typewriter-style echoes Teletext's data formatting.
Colour Language
Teal maps to Teletext cyan, gold to yellow, pink to magenta. The palette naturally aligns with the 8-colour Teletext system.

Accessibility & ADHD Principles
PLHub is designed to be ADHD-friendly first. That means high contrast, scannable content, and no cognitive overhead.
WCAG Targets
RequirementTargetNotes
Text contrast (normal)≥ 4.5:1 (AA)All body text against backgrounds
Text contrast (large)≥ 3:1 (AA)18px+ or 14px bold
Interactive contrast≥ 3:1 (AA)Buttons, links against backgrounds
Focus indicatorsVisible on all interactive elementsPink outline, 2px, offset 2px
Target size≥ 44×44px touch targets on mobileMinimum for buttons, links, toggles
MotionRespect prefers-reduced-motionDisable block reveal, pulse animations
ADHD-Friendly Design Constraints
High contrast text. The no-grey rule exists because of this. Navy or warm white — never washed out neutral tones that require extra effort to read.
Large, scannable fonts. Card headlines at 15px Sora 600. Body at 13px Sora 300. Nothing smaller than 10px. Line heights generous (1.35–1.55).
Bite-sized AI summaries. The core USP. Every story gets a 1-2 sentence summary. You get the gist without clicking through. The Pundit's Take is one more tap for colour — never mandatory.
No autoplay. No auto-playing video, audio, or animations that loop indefinitely. The LIVE pulse dot is the only continuous animation and it's 6px.
Clear visual hierarchy. Source → Headline → Summary → Tags. The eye path is the same on every card. Consistency reduces cognitive load.
No infinite scroll traps. "Load more stories" is a deliberate action. The user decides when to see more content.
Rewarding interactions. Every tap/click should feel premium — smooth transitions, gold accent for feedback. The site should feel like a nice place to be, not an anxiety machine.
Reduced Motion

Ad Policy
Brief and non-negotiable.
No gambling content. Ever.
No gambling ads, sponsorship, branded content, affiliate links, or odds integration. This is a hardcoded rule that applies to every placement on the site, in the app, on social media, and in any partnership. Monetisation strategy will be defined separately — this constraint is permanent regardless of what that strategy becomes.

Brand Rules
The definitive do and don't list. If you're unsure about a design decision, check here first.
Do
Use pink PL + white/dark HUB — always this weight split (700/300)
Keep gold restrained — icon bracket, pulse scores, and premium highlights only
Use teal for structural and interactive elements as defined in Section 06
Vary source labels by colour per feed (see Section 05)
Test all brand elements in both dark and light mode
Let the diagonal icon stand alone where space is tight
Use Sora 300 for body text — keep things light and readable
Use opacity on the base colour for text hierarchy — never grey hex codes
Use border + shadow on all cards in both modes
Use inline logo centred in nav on both mobile and desktop
Respect prefers-reduced-motion for all animations
Maintain 44×44px minimum touch targets on mobile
Don't
Never put gold text for "HUB" on a black or dark background (PornHub pattern)
Never complete the frame with four brackets — always two, diagonal only
Never use the wordmark without the diagonal icon nearby
Never rotate or angle the diagonal frame mark
No gambling content, branding, or sponsorship — ever
Never use pure black (#000000) as a background — always navy
Never use pink for large blocks of body text — headlines and accents only
Never use grey hex codes for text — use opacity on the base colour
Never use warm/cream white for light mode backgrounds — cool white only
Never autoplay video, audio, or infinite looping animations
Never use the stacked logo in the nav bar — inline only
Never use fonts other than Sora (except monospace for scores)
